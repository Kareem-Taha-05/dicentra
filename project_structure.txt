Architecture Overview
---------------------

Pattern: MVC (Model-View-Controller)

  в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ       в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ       в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
  в”Ӯ   GUI  (Views)  в”Ӯв”Җв”Җв”Җв”Җв”Җв”Җв–¶в”Ӯ  DicomController (C)  в”Ӯв”Җв”Җв”Җв”Җв”Җв”Җв–¶в”Ӯ DicomModel  в”Ӯ
  в”Ӯ  image_tab      в”Ӯв—Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ӯ  logic/controller.py  в”Ӯв—Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ӯ data layer  в”Ӯ
  в”Ӯ  metadata_tab   в”Ӯsignalsв”Ӯ                       в”Ӯ       в”Ӯ             в”Ӯ
  в”Ӯ  threed_tab     в”Ӯ       в”Ӯ  image_processor.py   в”Ӯ       в”Ӯ  pydicom    в”Ӯ
  в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ       в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ       в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

- Views emit user actions вҶ’ Controller methods
- Controller processes business logic вҶ’ emits Qt signals back to Views
- Model owns all DICOM I/O; no Qt in the data or logic layers
- Config layer centralises all constants; no magic numbers in source files
